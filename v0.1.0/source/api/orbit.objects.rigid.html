class omni.isaac.orbit.objects.rigid.RigidObject[source]#

Bases: object

Class for handling rigid objects.

Rigid objects are spawned from USD files and are encapsulated by a single root prim. The root prim is used to apply physics material to the rigid body.

This class wraps around RigidPrimView class from Isaac Sim to support the following:

  • Configuring using a single dataclass (struct).

  • Applying physics material to the rigid body.

  • Handling different rigid body views.

  • Storing data related to the rigid object.

Attributes:

objects

Rigid prim view for the rigid object.

cfg

Configuration class for the rigid object.

count

Number of prims encapsulated.

device

Memory device for computation.

data

Data related to articulation.

spawn(prim_path: str, translation: Sequence[float] | None = None, orientation: Sequence[float] | None = None)[source]#

Spawn a rigid object into the stage (loaded from its USD file).

Note

If inputs translation or orientation are not None, then they override the initial root state specified through the configuration class at spawning.

Parameters:

  • prim_path (str) – The prim path for spawning object at.

  • translation (Sequence[float], optional) – The local position of prim from its parent. Defaults to None.

  • orientation (Sequence[float], optional) – The local rotation (as quaternion (w, x, y, z) of the prim from its parent. Defaults to None.

initialize(prim_paths_expr: str | None = None)[source]#

Initializes the PhysX handles and internal buffers.

Note

PhysX handles are only enabled once the simulator starts playing. Hence, this function needs to be called whenever the simulator “plays” from a “stop” state.

Parameters:

prim_paths_expr (Optional[str], optional) – The prim path expression for the prims. Defaults to None.

Raises:

RuntimeError – When input prim_paths_expr is None, the method defaults to using the last prim path set when calling the spawn() function. In case, the object was not spawned and no valid prim_paths_expr is provided, the function throws an error.

Parameters:

update_buffers(dt: float | None = None)[source]#

Update the internal buffers.

The time step dt is used to compute numerical derivatives of quantities such as joint accelerations which are not provided by the simulator.

Parameters:

dt (float, optional) – The amount of time passed from last update_buffers call. Defaults to None.
